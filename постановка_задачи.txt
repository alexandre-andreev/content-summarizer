Я хочу сделать проект. Дизайн лежит в папке проекта. Пока делаем базовый сервис без авторизации и базы данных.
Вот описание проекта:
Я хочу разработать веб-сервис, который по ссылке на YouTube ролик будет формировать его краткое описание (транскрипт)

Веб-сервис должен включать серверную (back-end) и клиентскую (front-end) части c  использованием фрэймворка next.jx

Серверная и клиентская части должны работать на Vercel


Проект предполагает два этапа:


на первом этапе реализуется первая версия сервиса - работающий прототип без учета авторизации и без баз данных

на втором этапе реализуется вторая версия сервиса, будет добавлена база данных SupaBase и авторизация пользователей

Не первом этапе нужно сделать веб-страницу, на которой пользователю предоставляется возможность ввести URL адрес ролика YouTube

По нажатию пользователем кнопки "Получить описание" должно происходить обращение через API к сервису rapidai.com , который создаст и вернет транскрипт YouTube ролика. Транскрипт ролика отправляется в gemini api. Модель gemini flash 2.5. должна обработать этот транскрипт и сформировать готовое резюме, то есть краткое изложение содержания ролика.



Мне нужно чтобы для начала ты продумала архитектуру проекта. Архитектура должна соответствовать следующим требованиям:

#Безопасность

Необходимо обеспечить, чтобы API ключи не были видны пользователям. Для этого можно использовать серверную часть приложения, которая будет обрабатывать все API вызовы

Все ключи и конфиденциальные данные должны храниться в переменных окружения


#Логирование

В процессе разработке необходимо настроить логирование, чтобы можно было отслеживать ошибки и события

Логи должны сохраняться на сервере, их нужно будет просматривать через интерфейс пользователя или консоль


#Тестирование

На каждую функцию необходимо писать тесты, чтобы убедится в их корректной работе

Рекомендуется писать модульные тесты для поверки отдельных функций

#Развертывание.

все должно работать на Vercel, что обеспечит простоту развертывания и масштабируемость

необходимо настроить автоматические развертывание при при каждом commit в репозитории GitHub

Таким образом проект должен быть спроектирован с учетом безопасности, логирования и тестирования, чтобы обеспечить его надежную работу и простоту в обслуживании.
